# Card Wear Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cards visually age as rounds are played within a shoe, resetting fresh on each shuffle.

**Architecture:** A new `shoeRoundCount` Firebase field tracks rounds-since-shuffle. `renderTableState()` in `ui.js` reads it and applies a `wear-mid` or `wear-worn` class to `#table`. All visual effects are pure CSS descendant rules in `cards.css` — no per-card JS changes needed.

**Tech Stack:** Vanilla JS ES modules, Firebase RTDB, CSS custom properties, GitHub Pages.

```js
const shoeRound = room.shoeRoundCount ?? 0;
const wearStage = shoeRound >= 30 ? 'wear-worn' : shoeRound >= 10 ? 'wear-mid' : '';
const tableEl = document.getElementById('table');
if (tableEl) {
  tableEl.classList.remove('wear-mid', 'wear-worn');
  if (wearStage) tableEl.classList.add(wearStage);
```

## Task 3: Add wear CSS rules

**Files:**
- Modify: `css/cards.css` (append to end of file)

- [ ] **Step 1: Append wear rules to `css/cards.css`**

Add to the end of `css/cards.css`:

```css
/* Mid wear — slight yellowing, soft edge shadow */
#table.wear-mid .card-svg {
  filter: sepia(0.15) brightness(0.93);
  box-shadow: 2px 2px 0px 1px rgba(0,0,0,0.4), 3px 5px 12px rgba(0,0,0,0.6),
              inset 0 0 8px rgba(0,0,0,0.25);
}

/* Worn — visibly aged, heavy edge shadow, crease */
#table.wear-worn .card-svg {
  filter: sepia(0.35) brightness(0.87) contrast(0.95);
  box-shadow: 2px 2px 0px 1px rgba(0,0,0,0.4), 3px 5px 12px rgba(0,0,0,0.6),
              inset 0 0 14px rgba(0,0,0,0.45);
}

/* Crease — face-up cards only, worn stage only */
/* .card-wrap.face-down::after is already claimed by the shimmer animation */
#table.wear-worn .card-wrap:not(.face-down)::after {
  content: '';
  position: absolute;
  inset: 0;
  background: linear-gradient(
    128deg,
    transparent 42%,
    rgba(0,0,0,0.07) 50%,
    transparent 58%
  );
  pointer-events: none;
  border-radius: var(--radius-card);
}
```

- [ ] **Step 2: Verify in browser**

In Firebase console, set `shoeRoundCount` to 10. Confirm cards look slightly yellowed with edge darkening. Set to 30. Confirm stronger yellowing, heavier edge shadow, and a faint diagonal crease across face-up cards. Set back to 0. Confirm cards look clean again.

Also confirm:
- Face-down cards (dealer hole card) do NOT show the crease (the shimmer should still animate normally)
- Card hover lift still works
- Card sway animation still works
